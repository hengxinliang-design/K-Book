# K-Book 上传向导与文件管理页面流程设计

## 1. 文档目的

本文档定义 K-Book 第一期“上传分类命名”和“文件管理页面”的产品交互流程。

设计目标：

- 让使用者在文件进入知识库前完成分类、命名和元数据确认。
- 让文件上传后的目录、标签和业务元数据可以被持续管理。
- 保持 Open Notebook 现有 Source、Notebook、Reference、解析和向量化能力稳定。
- 不把客服工单、审批流程、项目实施流程或 Podcast 能力带入 K-Book。

本文档只定义产品流程和服务边界，不定义最终 UI 视觉稿。

## 2. 适用范围

第一期覆盖：

- 知识库内上传文件。
- 批量上传文件。
- 上传前选择知识库和目录。
- 上传前确认文件显示名称。
- 上传前设置标签、模块、文档类型、业务版本和状态。
- 文件管理页面的目录浏览、列表查看、筛选、排序和元数据编辑。
- 将已有 Source 加入知识库并选择目录。
- 从知识库移除文件关联。

第一期不覆盖：

- 文件替换和重新学习；该能力进入第二期。
- 版本历史比较。
- 审批入库流程。
- 用户、角色和项目权限。
- Podcast 生成、展示和配置。

## 3. 页面信息架构

K-Book 第一期新增或调整以下页面区域：

```text
知识库列表
└─ 知识库详情
   ├─ 问答/检索
   ├─ 文件管理
   │  ├─ 目录树
   │  ├─ 文件列表
   │  ├─ 筛选器
   │  └─ 文件详情侧栏
   └─ 知识库设置
      ├─ 基本信息
      ├─ 客户/项目/LN 版本/适用范围
      └─ 目录和字典入口

系统管理
└─ 字典管理
   ├─ 标签
   ├─ LN 模块
   ├─ 文档类型
   ├─ 文档状态
   └─ LN 版本
```

文件上传入口：

- 知识库详情页的“上传文件”按钮。
- 文件管理页工具栏的“上传文件”按钮。
- 空目录状态中的“上传到当前目录”按钮。

不提供：

- 主导航 Podcast 入口。
- 快速创建 Podcast 入口。
- 从文件列表触发 Podcast 的操作。

## 4. 上传向导总流程

目标流程：

```text
选择文件
→ 选择位置
→ 设置批量默认值
→ 逐文件确认
→ 确认提交
→ 上传、解析和学习
→ 查看处理结果
```

对应底层写入：

```text
source
reference(source -> notebook, folder)
source_profile
source_tag
```

上传向导必须把“文件内容处理”和“业务分类确认”分离。文件未提交前，只保存在浏览器待上传队列中；确认提交后才创建 Source 和相关关系。

## 5. 上传向导步骤

### 5.1 选择文件

使用者选择一个或多个文件。

上传区域必须展示灰色辅助备注，明确当前允许上传的资料格式。

推荐呈现：

```text
拖拽文件到这里，或点击上传
PDF、Word、PPT、Excel、文本、Markdown、网页、图片、音视频、压缩包等
支持格式：pdf、doc、docx、ppt、pptx、xls、xlsx、txt、md、epub、html、jpg、png、mp3、mp4 等
```

展示要求：

- 使用 `muted` 灰色文字，作为说明性信息，不抢占主操作视觉层级。
- 简短格式摘要直接展示在上传区域内。
- 完整扩展名清单可以通过“支持格式”文本悬浮提示、帮助图标或展开链接查看。
- 完整清单较长时不直接铺满页面，避免干扰上传主流程。
- 格式说明必须和实际前后端校验共用同一份配置或由后端接口返回，不能在 UI 中硬编码另一套清单。

第一期初始支持格式按当前 Open Notebook 前端上传白名单对齐：

```text
pdf, doc, docx, ppt, pptx, xls, xlsx, txt, md, epub, html,
mp4, avi, mov, wmv, mp3, wav, m4a, aac,
jpg, jpeg, png, tiff,
zip, tar, gz
```

界面展示：

| 字段 | 说明 |
| --- | --- |
| 原文件名 | 从本地文件读取 |
| 文件大小 | 用于判断是否超出限制 |
| 文件类型 | 用于判断是否支持 |
| 初始显示名称 | 默认由原文件名去扩展名生成 |
| 待处理状态 | 待确认、不可上传、重复提示等 |

校验规则：

- 不允许空文件。
- 不允许超出系统配置大小的文件。
- 不允许不支持的文件类型进入下一步。
- 不支持的文件类型需要在文件行内显示错误，并提示查看支持格式。
- 同一批次中原文件名重复时不阻止上传，但必须提示使用者确认显示名称。
- 文件显示名称允许重复，但同一知识库内重复时应提示，避免使用者误判。

失败边界：

- 本地文件读取失败：该文件标记为不可上传，其他文件继续。
- 批次中所有文件都不可上传：禁用下一步。

### 5.2 选择位置

使用者选择目标知识库和目录。

入口来自知识库详情页时：

- 知识库默认锁定为当前知识库。
- 目录默认使用当前目录；如果从知识库根目录进入，则目录为空。

入口来自全局上传时：

- 必须先选择知识库。
- 选择知识库后再加载该知识库目录树。

目录选择规则：

- 根目录使用空值表示，对应 `reference.folder = NONE`。
- 目录树只展示当前知识库的 Folder。
- 不允许选择其他知识库的目录。
- 如果目录已被删除，提交前必须重新选择。

### 5.3 设置批量默认值

批量默认值用于减少重复填写。

可设置字段：

| 字段 | 来源 | 写入位置 |
| --- | --- | --- |
| 目录 | 当前知识库目录树 | `reference.folder` |
| 标签 | `dictionary_item(tag)` | `source_tag` |
| 模块 | `dictionary_item(module)` | `source_profile.module` |
| 文档类型 | `dictionary_item(document_type)` | `source_profile.document_type` |
| 业务版本 | 手工输入 | `source_profile.business_version` |
| 状态 | `dictionary_item(document_status)` | `source_profile.status` |

默认行为：

- 新加入批次的文件继承当前批量默认值。
- 已经逐文件修改过的字段不被后续批量默认值覆盖，除非使用者选择“覆盖全部文件”。
- 标签默认采用追加模式；使用者可选择替换全部标签。

建议第一期提供三个快捷设置：

1. 应用到未修改文件。
2. 覆盖全部文件。
3. 清空全部文件某字段。

### 5.4 逐文件确认

使用者逐文件确认最终名称和元数据。

文件表格字段：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 显示名称 | 是 | 写入 `source.title` |
| 目录 | 否 | 写入 `reference.folder` |
| 标签 | 否 | 写入 `source_tag` |
| 模块 | 否 | 写入 `source_profile.module` |
| 文档类型 | 否 | 写入 `source_profile.document_type` |
| 业务版本 | 否 | 写入 `source_profile.business_version` |
| 状态 | 否 | 写入 `source_profile.status` |
| 原文件名 | 只读 | 写入 `source_profile.original_filename` |

显示名称规则：

- 默认值为去扩展名后的原文件名。
- 前后空格自动清理。
- 不允许为空。
- 长度超过限制时必须阻止提交。
- 不把显示名称作为物理文件名。

字典字段规则：

- 只能选择 active 字典项。
- 停用项可以在历史文件中展示，但不能在上传时新增选择。
- 字典项加载失败时，相关字段不可编辑，不能提交依赖该字段的新值。

### 5.5 确认提交

确认页必须展示最终会写入的业务信息。

按文件展示：

- 显示名称。
- 目标知识库。
- 目录路径。
- 标签。
- 模块。
- 文档类型。
- 业务版本。
- 状态。
- 原文件名。
- 文件大小。

批次级提示：

- 本次上传文件总数。
- 可上传数量。
- 不可上传数量。
- 是否存在显示名称重复。
- 是否存在未填写关键元数据的文件。

关键元数据不强制必填，但需要提示：

- 模块为空。
- 文档类型为空。
- 状态为空。

原因：实施项目资料历史来源复杂，第一期不能因为元数据不完整阻断资料入库。

### 5.6 上传、解析和学习

提交后按文件独立处理。

建议状态：

| 状态 | 说明 |
| --- | --- |
| queued | 已提交，等待上传或处理 |
| uploading | 文件上传中 |
| uploaded | 文件已上传，等待解析 |
| processing | 正在解析、分块、生成向量 |
| ready | 可检索、可问答 |
| failed | 上传或处理失败 |

批量上传事务边界：

- 不要求整个批次同一事务。
- 每个文件独立创建 Source、Reference、SourceProfile 和 SourceTag。
- 一个文件失败不回滚其他文件。
- 文件自己的业务关系写入必须与 Source 创建保持一致；不能出现 Source 已创建但缺少必要 Reference 的半成品。

失败处理：

- 上传失败：不创建 Source。
- Source 创建失败：该文件失败，允许重试。
- Reference 创建失败：必须回滚该文件 Source 或标记为不可用并进入补偿队列。
- SourceProfile 或 SourceTag 创建失败：该文件不得进入 ready 状态，必须允许重试业务信息写入。
- 内容解析失败：保留 Source 和业务元数据，但状态为 failed；第一期允许删除后重新上传，第二期提供重新学习。

### 5.7 查看处理结果

上传结果页按文件展示：

- 成功文件数量。
- 失败文件数量。
- 每个文件最终状态。
- 失败原因。
- 操作入口：查看文件、进入目录、重试失败项、删除失败项。

批次完成后：

- 成功文件立即出现在文件管理页。
- 处理中的文件可见，但标记为 processing。
- failed 文件默认可见，避免使用者重复上传而不知道失败记录存在。

## 6. 文件管理页面布局

推荐布局：

```text
┌────────────────────────────────────────────────────────────┐
│ 知识库：客户 / 项目 / LN版本 / 适用范围                    │
├───────────────┬────────────────────────────────────────────┤
│ 目录树         │ 工具栏：上传 / 新建目录 / 批量操作 / 搜索   │
│               ├────────────────────────────────────────────┤
│ 全部文件       │ 筛选器：标签 / 模块 / 类型 / 状态 / 版本     │
│ 根目录         ├────────────────────────────────────────────┤
│ ├─ 需求        │ 文件列表                                    │
│ ├─ 蓝图        │ 名称 | 目录 | 标签 | 模块 | 类型 | 版本 | 状态 │
│ └─ 技术设计    ├────────────────────────────────────────────┤
│               │ 文件详情侧栏                                │
└───────────────┴────────────────────────────────────────────┘
```

### 6.1 顶部上下文

必须显示当前知识库上下文：

- 知识库名称。
- 客户。
- 项目。
- LN 版本。
- 适用范围。

目的：避免客服或顾问在错误项目、错误客户或错误 LN 版本下查资料。

### 6.2 目录树

目录树能力：

- 查看全部文件。
- 查看根目录文件。
- 查看某个目录下文件。
- 新建目录。
- 重命名目录。
- 移动目录。
- 删除空目录。

目录删除规则：

- 空目录：允许删除。
- 非空目录：第一期默认阻止删除，并提示先移动或删除文件和子目录。
- 后续可增加“级联删除目录结构但不删除 Source”的显式高级操作。

目录移动规则：

- 不允许移动到自身。
- 不允许移动到自身后代。
- 不允许跨知识库移动目录。
- 同级重名时阻止。

### 6.3 文件列表

列表字段：

| 字段 | 来源 |
| --- | --- |
| 显示名称 | `source.title` |
| 目录 | `reference.folder` |
| 标签 | `source_tag -> dictionary_item(tag)` |
| 模块 | `source_profile.module` |
| 文档类型 | `source_profile.document_type` |
| 业务版本 | `source_profile.business_version` |
| 状态 | `source_profile.status` |
| 处理状态 | Source 或处理任务状态 |
| 更新时间 | `source.updated` 或 `source_profile.updated` |

第一期排序：

- 默认按更新时间倒序。
- 支持按名称、文档类型、状态、业务版本排序。

第一期筛选：

- 目录。
- 标签，多选。
- 模块。
- 文档类型。
- 状态。
- 业务版本关键字。
- 文件名称关键字。
- 处理状态。

筛选组合规则：

- 不同筛选字段之间使用 AND。
- 标签多选默认使用 OR，即包含任一标签。
- 后续可增加“必须同时包含所有标签”的高级模式。

### 6.4 文件详情侧栏

点击文件行后打开详情侧栏。

展示：

- 显示名称。
- 原文件名。
- 所属知识库。
- 所在目录。
- 标签。
- 模块。
- 文档类型。
- 业务版本。
- 状态。
- 处理状态。
- 加入知识库时间。
- 最近更新时间。

操作：

- 编辑显示名称。
- 移动目录。
- 编辑标签。
- 编辑模块、文档类型、业务版本和状态。
- 从当前知识库移除。
- 打开问答或检索入口。

第一期不提供：

- 替换文件。
- 重新学习。
- 查看版本历史。

这些入口可以在 UI 上预留位置，但必须禁用或不展示，避免误导。

## 7. 关键操作流程

### 7.1 新建目录

```text
选择父目录
→ 输入目录名称
→ 校验同级唯一
→ 创建 Folder
→ 刷新目录树
```

边界：

- 根目录下同名阻止。
- 同一父目录下同名阻止。
- 不同目录下允许同名。
- 目录名称前后空格自动清理。

### 7.2 移动文件

```text
选择一个或多个文件
→ 选择目标目录
→ 校验目录属于当前知识库
→ 更新 reference.folder
→ 刷新列表
```

影响：

- 不修改 Source。
- 不修改 SourceProfile。
- 不修改 SourceTag。
- 不触发重新学习。

### 7.3 编辑文件元数据

```text
打开文件详情
→ 修改模块/类型/版本/状态/标签
→ 校验字典项 active 且类型正确
→ 更新 source_profile 和 source_tag
→ 刷新列表与详情
```

影响：

- 不触发解析。
- 不重建向量。
- 检索范围筛选立即生效。

### 7.4 从知识库移除文件

```text
选择文件
→ 点击从知识库移除
→ 二次确认
→ 删除 reference(source -> notebook)
→ 文件从当前知识库列表消失
```

边界：

- 只移除当前知识库关联，不一定删除 Source。
- 如果 Source 不再关联任何知识库，是否物理删除原 Source 由后续清理策略决定。
- 移除后不影响其他知识库中同一 Source 的目录位置。

### 7.5 添加已有 Source 到知识库

第一期可以保留“上传新文件”为主路径；如果提供“添加已有文件”能力，必须按以下流程：

```text
搜索已有 Source
→ 选择目标知识库和目录
→ 可调整显示名称、标签和元数据
→ 创建 reference
→ 更新 SourceProfile/SourceTag
```

边界：

- 同一 Source 与同一 Notebook 已有关联时，不重复创建 Reference。
- SourceProfile 和 SourceTag 是 Source 全局属性，修改会影响其他知识库对该 Source 的展示。
- UI 必须提示“这些元数据是文件全局属性”。

## 8. 数据一致性规则

### 8.1 Source、Reference 和 Profile

上传成功后的最小一致状态：

```text
source 存在
reference(source -> notebook) 存在
source_profile 可以为空或存在
source_tag 可以为空或多条
```

如果使用者填写了任一业务元数据，则必须创建或更新 SourceProfile。

### 8.2 字典项

写入规则：

- 上传和编辑只能使用 active 字典项。
- 停用字典项保留历史展示。
- 删除已被引用的字典项必须被业务层阻止。

### 8.3 目录

写入规则：

- `reference.folder = NONE` 表示根目录。
- `reference.folder` 非空时，该 Folder 必须属于同一 Notebook。
- 移动目录不能导致循环。

### 8.4 批量操作

批量操作原则：

- 先整体校验用户输入。
- 再按文件逐个提交。
- 返回每个文件的成功或失败结果。
- 不因单个文件失败回滚整个批次。

## 9. 空状态和异常状态

### 9.1 空知识库

展示：

- 当前知识库尚无文件。
- 主操作：“上传文件”。
- 次操作：“新建目录”。

### 9.2 空目录

展示：

- 当前目录暂无文件。
- 主操作：“上传到当前目录”。
- 次操作：“移动已有文件到此目录”。

### 9.3 字典为空

如果模块、文档类型、状态或标签字典为空：

- 上传仍可继续。
- 对应字段显示为空状态。
- 管理员可进入字典管理补充。

原因：不能因为字典未初始化阻断历史资料导入。

### 9.4 文件处理中

处理中文件：

- 在列表中可见。
- 可查看业务元数据。
- 不允许问答引用该文件内容。
- 允许编辑目录、标签和元数据。
- 不允许删除底层处理中任务；如删除文件关联，需要提示处理结果可能仍会产生 Source 记录，后续由清理任务处理。

### 9.5 处理失败

失败文件：

- 在列表中可见。
- 展示失败原因。
- 可编辑业务元数据。
- 可从知识库移除。
- 第一期建议提供“删除失败记录”。
- “重新学习”进入第二期。

## 10. 权限预留

第一期不实现用户角色权限，但页面和 API 设计不能阻塞后续扩展：

- 所有写操作从服务层进入，不在前端直接拼接数据库操作。
- 文件列表查询必须带 Notebook 上下文。
- 文件移动、目录管理和元数据编辑必须保留操作者上下文参数。
- 后续可以在 Notebook、Project、Folder、Source 或 Reference 维度增加授权判断。

## 12. 验收标准

上传向导验收：

- 单文件上传可以指定知识库、目录、显示名称、标签和业务元数据。
- 批量上传可以设置公共默认值，并逐文件覆盖。
- 上传确认页准确展示最终写入信息。
- 单个文件失败不影响其他文件。
- 上传成功后文件出现在正确目录。
- 元数据修改不触发重新学习。

文件管理验收：

- 使用者可以通过目录树浏览文件。
- 使用者可以按标签、模块、文档类型、状态和名称筛选文件。
- 使用者可以移动文件目录。
- 使用者可以编辑文件标签和业务元数据。
- 使用者可以从当前知识库移除文件，且不影响其他知识库中的同一 Source。
- 处理中文件和失败文件有明确状态。

边界验收：

- 不能选择其他知识库的目录。
- 同一 Source 与同一 Notebook 不重复创建 Reference。
- 停用字典项不能用于新上传。
- 空字典不阻止上传。
- 非空目录默认不能删除。
- 页面不出现 Podcast 正常入口。
